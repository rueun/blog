    JPA 엔티티 매핑 - 연관관계 설정과 영속성 컨텍스트

    2026. 02. 25. 00:00 · 읽는데 1분

    JPA란?#

    JPA(Java Persistence API)는 자바 ORM 기술의 표준 인터페이스입니다.

    엔티티 매핑#

    JAVA
     1@Entity
 2@Table(name = "members")
 3public class Member {
 4
 5    @Id
 6    @GeneratedValue(strategy = GenerationType.IDENTITY)
 7    private Long id;
 8
 9    @Column(nullable = false, length = 50)
10    private String name;
11
12    @ManyToOne(fetch = FetchType.LAZY)
13    @JoinColumn(name = "team_id")
14    private Team team;
15}

    연관관계#

    다대일 (ManyToOne)#

    가장 많이 사용하는 연관관계입니다. 외래 키가 있는 쪽이 연관관계의 주인입니다.

    일대다 (OneToMany)#

    JAVA
     1@Entity
 2public class Team {
 3
 4    @Id
 5    @GeneratedValue(strategy = GenerationType.IDENTITY)
 6    private Long id;
 7
 8    private String name;
 9
10    @OneToMany(mappedBy = "team", cascade = CascadeType.ALL)
11    private List<Member> members = new ArrayList<>();
12}

    영속성 컨텍스트#

    • 1차 캐시: 같은 트랜잭션 내에서 동일한 엔티티 보장
    • 변경 감지(Dirty Checking): 엔티티 수정 시 자동 UPDATE
    • 지연 로딩(Lazy Loading): 실제 사용 시점에 쿼리 실행

    정리#

    JPA의 연관관계와 영속성 컨텍스트를 이해하면 효율적인 데이터 접근 계층을 구현할 수 있습니다.
